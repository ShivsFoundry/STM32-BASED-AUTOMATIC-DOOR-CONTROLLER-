# Security Policy

Thank you for helping improve the security of this project.

## Supported Versions

The latest version of this repository is actively maintained and receives security-related updates.

| Version | Supported |
|---------|-----------|
| Latest | ✅ Yes |
| Older Releases | ❌ No |

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly instead of disclosing it publicly.

When reporting an issue, include:

- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Suggested fix (if available)

Please open a **private GitHub Security Advisory** or contact the repository maintainer directly.

---

## Response Process

After receiving a valid security report, the following process will be followed:

1. Acknowledge the report.
2. Investigate the reported issue.
3. Develop and test a fix if necessary.
4. Release an updated version of the project.
5. Credit the reporter when appropriate.

---

## Scope

This policy applies to:

- Source code
- Documentation
- Project configuration files
- Repository resources

It does not cover issues caused by third-party hardware, external libraries, or user modifications outside this repository.

---

## Responsible Disclosure

Please do not publicly disclose security issues until they have been reviewed and addressed.

Responsible disclosure helps protect users and allows sufficient time to develop a proper fix.

---

Thank you for helping improve the security and reliability of this project.
